1. What is a Dummy Circuit Breaker?

A dummy circuit breaker is used for relay testing, when a realistic representation of a high voltage circuit breaker is required.

Normally when testing a protection relay with an Omicron test set (or similar), the test set injects fault current and times how long the relay takes to assert its trip output, and then immediately stops the injected current. Modern relays are becoming more 'clever' and in some cases, they will trigger an error or behave unexpectedly when they see the fault current stopped more quickly than expected. A real circuit breaker takes 50–150 ms to open after receiving a trip command.

A dummy circuit breaker is also used when testing some functionaity such as auto reclose, where the state of the circuit breaker needs to remain consistent between tests, like a real circuit breaker will.

This SEL-551 is configured to simulate a real circuit breaker by acting as a latching relay, and implementing breaker opening and closing delay. The relay-under-test and test set see it as a real circuit breaker.

Trip (IN1 → OUT1 opens)
Receives trip command → waits 100 ms → opens 52A contact
Close (IN2 → OUT1 closes)
Receives close command → waits 100 ms → closes 52A contact

3. Wiring

3.1 Overview

3.2 Control Power

Connect DC control power to the relay power supply terminals per the relay nameplate. Observe correct polarity.

3.3 IN1 — Trip Command Input

IN1 is wired so that when the relay-under-test closes its trip output contact, IN1 asserts and the dummy breaker starts its 100 ms opening timer.

3.4 IN2 — Close Command Input

Connect one or more of the following in parallel to IN2:

  • Relay-under-test CLOSE output (if autorecloser or close output exists)
  • Test set CLOSE output / timing channel output
  • External push button (normally open, momentary or maintained)

3.5 OUT1 — 52A Contact (Primary Output)

Normally open dry contact. CLOSED (makes) when the breaker is CLOSED; OPEN (breaks) when the breaker is OPEN.

On the connectorised SEL-551, OUT1 is polarity-dependent. Terminal A02 must be (+) relative to A01.

3.6 OUT2 — 52B Contact (Optional)

Complement of OUT1. CLOSED when breaker is OPEN. Connect to the relay-under-test's 52B input if required.

Power-On State

The TRIP latch initialises to 0. The dummy breaker starts in the CLOSED state.

Trip Test Sequence

The test set should monitor IN1 as the "breaker open" signal as this will ensure it is measuring the actual relay timing, excluding the breaker delay.

Resetting After a Trip Test

  1. If the Relay has a latching trip output, reset the Relay first so IN1 de-asserts. (See Section 6 — Anti-Pumping.)
  2. Assert IN2 — press close button, or have the test set assert the close command.
  3. After 100 ms the breaker closes and the display returns to the normal state.

If IN2 is unavailable, press TARGET RESET on the SEL-551 front panel. This directly unlatches the TRIP bit and closes the breaker immediately — no 100 ms close delay. Use for test reset only.

6. Anti-Pumping Behaviour

The dummy breaker will not accept a close command (IN2) while a trip command (IN1) is simultaneously asserted. This mirrors real breaker anti-pumping protection.

Anti-pump condition
CB OPEN
ANTI PUMP

Anti-pumping is enforced at two levels:

  • The close timer can only run when TRIP = 1 and IN1 = 0.
  • The unlatch equation ULTR = SV5T * !IN1 cannot fire while IN1 is held.
Practical: If the Relay holds its trip output latched after a fault, reset the Relay before attempting to close the dummy breaker.

9. Adjusting the Time Delay

A single setting — SV5PU in SET_1.TXT — controls both the opening and closing delay. They are always equal.

All timers are in cycles (1/8-cycle resolution). At 50 Hz: 1 cycle = 20 ms.

Desired delaySV5PU value
50 ms2.500
83 ms4.125
100 ms (default)5.000
150 ms7.500
200 ms10.000

Method A — Via QuickSet or Serial Terminal

  1. Change SV5PU in SET_1.TXT to the desired cycle count.
  2. Send updated settings to the relay (groups 1 and L).
No display label update is required — the bottom row always shows OP/CL DLY SV5PU, which refers to the setting name and remains accurate at any timer value.

Method B — Via Relay Front-Panel Menu

Settings can be changed directly from the relay front panel without a laptop:

  1. Press SET (or navigate the front-panel menu to Settings → Group 1).
  2. Press the right-arrow key to scroll through settings until SV5PU appears on the display.
  3. Press SELECT (or ENTER). The current value is shown and the cursor becomes active.
  4. Use the arrow keys to change the value to the desired cycle count (see table above).
  5. Press SELECT to confirm. The relay will prompt for the access-level 2 password if not already logged in.
  6. Confirm the new value is shown on the display.
Note: After changing settings from the front panel, verify the change by reading back the setting. Some relay firmware versions require pressing END to save and exit the settings edit mode.

10. Front-Panel Local Controls

There are two front-panel local control switches — MAN TRIP (LB1) and MAN CLOSE (LB2) — so the breaker can be tripped or closed from the relay front panel without any external wiring.

LB1 and LB2 are configured as momentary (type M) switches. Pressing SELECT briefly pulses the local bit for one processing interval, then it automatically returns. This is equivalent to a momentary push button press.

Accessing Local Controls

  1. Press the CNTRL pushbutton on the relay front panel. The display shows:
    Press CNTRL for
    Local Control
  2. Press CNTRL again. The first local control switch is shown:
    MAN TRIP
    RETURN
  3. Press RIGHT ARROW to scroll to the next switch (MAN CLOSE).
  4. When the desired switch is shown, press SELECT. The display shows the operate prompt:
    To trip the breaker
    MAN TRIP
    No Yes
    To close the breaker
    MAN CLOSE
    No Yes
  5. Press LEFT ARROW to move the cursor to Yes.
  6. Press SELECT to execute. The local bit fires momentarily and the breaker begins its 100 ms operating sequence.

11. Spring Charge Simulation

Real HV circuit breakers use a spring-charged operating mechanism. After a Close–Open (CO) operation, the spring must recharge before the breaker can close again. The standard operating sequence for many HV breakers is:

O — 0.3 s — CO — 15 s — CO
SymbolMeaning
OOpen (trip). The breaker opens.
0.3 sDead time — minimum wait before first reclose attempt (arc de-ionisation time).
COClose then immediate Open. The breaker closes (reclose attempt) and, if the fault is still present, re-trips. This consumes one spring charge.
15 sSpring recharge time — the motor must recharge the spring before the next close.
COSecond reclose attempt. Final close; if it re-trips again the breaker locks out.

How the Dummy Breaker Simulates This

TimerSettingFunction
SV6PU15 cycles (0.3 s)Dead-time timer. Starts when TRIP=1. Close is blocked until SV6T=1.
SV7DO50 cycles (1 s)Reclose memory window. SV7T=1 for 1 s after the breaker closes. Used to detect a CO event (re-trip within 1 s of a close).
SV8DO750 cycles (15 s)Spring charge block. SV8T=1 for 15 s after a CO event. Close is blocked while SV8T=1. OUT3 (SPRING CHG lamp) energises.

O-0.3s-CO-15s-CO Timing Sequence

Operating Rules

  • First close attempt: Only allowed ≥ 0.3 s after the breaker opened (SV6T=1). Assert IN2 or press MAN CLOSE.
  • Spring charge starts: If the fault is still present and the RUT re-trips within 1 s of the close, SV8T asserts and blocks the close path for 15 s. OUT3 (SPRING CHG lamp) illuminates.
  • Second close attempt: Allowed once SV8T drops (OUT3 goes off) — and also once 0.3 s has elapsed since the re-trip (which will always be satisfied by then).
  • Third trip without close: If the second reclose also re-trips, the breaker stays open. SV8 triggers again for another 15 s. There is no lockout in V0.6 — the sequence can repeat.
Testing tip: For tests that do not require spring charge simulation, the behaviour is transparent — the 0.3 s dead time and 15 s block only activate if there is a CO event. A simple O → IN2 close sequence (fault clears before reclose) bypasses both timers.

13. Limitations and Notes

#Note
1 No CT/VT connections. Do not connect current or voltage transformers while the relay is configured as a dummy breaker. All overcurrent protection elements are disabled.
2 Hardware contact delay (~8 ms). Output relay contacts have a hardware operate time of approximately 8 ms in addition to the programmed timer. Total operating time ≈ 108 ms at the 100 ms setting. This is consistent with real breaker behaviour and is measurable via the SER.
3 Minimum trip duration (TDURD = 3 cycles / 60 ms). The TRIP latch will not unlatch less than 60 ms after it sets. In practice this never constrains close timing since the close timer adds 100 ms on top.
4 Power-loss behaviour. If control power is lost, all output contacts de-energise. OUT1 (52A) opens — the Relay sees the breaker as OPEN. Fail-open / fail-safe.
5 Input debounce (0.25 cycle / ~5 ms). The SEL-551 optoisolated inputs have a built-in 0.25-cycle debounce filter. Transients shorter than 5 ms will not be seen by the relay.
6 Opening and closing delays are equal. A single timer (SV5PU) governs both. The dead-time (0.3 s) and spring-charge (15 s) timers are separate — only SV5PU affects operating time. If independent open and close delays are required, refer to the design plan.
